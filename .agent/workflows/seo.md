# /seo — 構造最適化

各ページのSEO・アクセシビリティ・ナビゲーション構造を分析し、改善する。

## 手順

1. 全 `.md` ファイルの見出し階層を検証する:
   - 各ファイルに `h1` が1つだけ存在するか
   - `h2` → `h3` → `h4` の階層がスキップされていないか
   - 見出しが意味的に適切か

// turbo
2. `mkdocs.yml` の `nav` と実ファイルの整合性を確認する:
   ```bash
   find docs/ -name "*.md" -type f | sort
   ```
   上記結果と `nav` に登録されたファイルを照合し、未登録ファイルを検出する。

3. リンクグラフを生成する:
   - 各ページの内部リンクを抽出し、リンク先の存在を確認
   - 被リンクが0のページ（孤立ページ）を検出
   - デッドエンドページ（外部リンクを除き、内部リンクが0のページ）を検出

4. MkDocs Material の機能活用状況を確認する:
   - `admonition` (注意書き) が適切に使われているか
   - `tabs` が活用できる箇所がないか
   - `attr_list` でカード画像に適切な属性が付与されているか

5. 改善提案を `notify_user` でユーザーに提示する。

6. 承認後、修正を適用する。

// turbo
7. `task build` でビルド検証を実行する。
