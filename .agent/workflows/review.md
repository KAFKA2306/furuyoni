# /review — コンテンツレビュー

`furuyoni_reviewer` スキルの3ペルソナ（Hajime/Kenshin/Ruri）を全適用し、対象ファイルのレビュー → 修正案を生成する。

## 手順

1. `.agent/skills/furuyoni_reviewer/SKILL.md` を読み込み、ペルソナ定義を把握する。

2. 対象ファイルを特定する。
   - 引数でファイルパスが指定されていればそのファイル
   - 未指定なら `docs/megami/*.md` の全ファイル
   - ユーザーが開いているアクティブドキュメントがあればそれを優先

3. 各ペルソナで順次レビューを実行する:
   - **Hajime（初心者）**: 用語の不明瞭さ、レイアウトの圧迫感、カードとルールの接続性
   - **Kenshin（戦術家）**: シナジーの妥当性、デッキ構築アドバイスの精度、戦略的深度
   - **Ruri（ルール審判）**: ルール準拠、タイミング記述、キーワード正確性

4. 指摘を統合し、重複排除・優先度付けを行う。

5. 修正案を **diff形式** で出力する。各提案は以下の形式:
   ```
   ### 指摘 #N: [タイトル]
   - ペルソナ: [Hajime/Kenshin/Ruri]
   - 重要度: [高/中/低]
   - 原文: `対象テキスト`
   - 修正案: `修正後テキスト`
   - 理由: 修正の根拠
   ```

// turbo
6. `notify_user` で修正案をユーザーに提示し、承認を求める。

7. 承認後、`replace_file_content` または `multi_replace_file_content` で修正を適用する。

8. 適用後、`task build` でビルド検証を実行する。
// turbo
